Anda adalah seorang pengembang perangkat lunak profesional dengan keahlian dalam tumpukan **PHP, Bootstrap, dan MySQL**. Saya ingin Anda merancang dan membuat aplikasi **Pengelolaan Stok Barang (Inventori) dan Sistem Kasir (Point of Sale/POS)** yang terintegrasi.

**1. Teknologi dan Arsitektur:**
* **Backend & Logika:** PHP Native atau framework PHP (misalnya, **Laravel atau CodeIgniter**) - *Pilih salah satu:* [PHP Native / Laravel / CodeIgniter].
* **Database:** MySQL.
* **Frontend & Styling:** HTML, CSS, JavaScript, dan **Bootstrap** untuk desain responsif.
* **Arsitektur:** Aplikasi Web berbasis server (Multi-Page Application) dengan pemisahan MVC (Model-View-Controller) jika menggunakan framework.

**2. Fitur Utama yang Diperbarui:**

| Kategori Menu | Fitur Inti yang Diperlukan | Integrasi dengan Inventori |
| :--- | :--- | :--- |
| **1. Dashboard** | Ringkasan Stok, Grafik Penjualan Harian, Peringatan Stok Minimum. | Menarik data *real-time* dari tabel Stok dan Transaksi Penjualan/Keluar. |
| **2. Kelola Barang** | CRUD Barang (SKU/Barcode, Nama, Kategori, Harga Beli, Harga Jual, Satuan, Stok Saat Ini). | **INTI:** Data harga jual dan stok digunakan oleh sistem kasir. |
| **3. Kelola Supplier & Pelanggan** | CRUD data Master Supplier dan Pelanggan. | Digunakan dalam Transaksi Masuk (Supplier) dan Transaksi POS (Pelanggan). |
| **4. Transaksi Masuk (Pembelian)** | Formulir untuk mencatat barang masuk dari Supplier. | **MENAMBAH** stok barang di tabel Items. |
| **5. Sistem Kasir (POS)** | **Antarmuka khusus kasir (tampilan cepat/tablet)**: Pilih Barang berdasarkan nama/barcode, Input Kuantitas, Hitung Subtotal, Diskon, Total Bayar, Uang Pembayaran, Hitung Kembalian. | **MENGURANGI** stok barang secara **otomatis** setelah transaksi sukses. |
| **6. Laporan** | Laporan Stok Saat Ini, **Laporan Penjualan (Harian/Bulanan)**, Laporan Laba Kotor (dihitung dari Harga Jual - Harga Beli). | Memberikan analisis keuangan dari aktivitas POS dan Stok. |

**3. Output yang Saya Inginkan:**
1.  **Skema Database MySQL:** Berikan skema DDL (Data Definition Language) SQL untuk tabel utama (misalnya: `users`, `items`, `suppliers`, `customers`, `transactions_header`, `transaction_details`). Pastikan ada **foreign key** yang menghubungkan `transaction_details` ke `items`.
2.  **Struktur Folder PHP:** Tunjukkan struktur folder yang disarankan untuk aplikasi PHP-MySQL (misalnya: `config`, `views`, `controllers`, `models`, `assets`).
3.  **Contoh Kode:** Berikan contoh kode untuk **Logika PHP Controller/Function** yang menangani proses **simpan transaksi kasir** dan **mengurangi stok** secara atomik (dalam satu blok operasi). Gunakan sintaks PDO untuk koneksi database.

**Tujuan:** Hasil harus merupakan kerangka aplikasi yang kokoh, terintegrasi penuh antara Inventori dan POS, dan siap dikembangkan menggunakan PHP/Bootstrap/MySQL.
